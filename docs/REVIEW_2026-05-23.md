# PocketLog – Multi-Agent Review Report
**Stand: 23. Mai 2026**

Dieser Report wurde aus den Einzelbefunden von 8 spezialisierten Review-Agents synthetisiert
(code-review, security-review, ui-review, db-review, token-audit, copy-review, pwa-review, vendor-audit)
und von einem Projektleiter-Agent konsolidiert.

**Gesamtzahl Issues: 6 P0 · 10 P1 · 22 P2**

---

## Executive Summary

PocketLog ist ein solides, gut strukturiertes Projekt mit klarer Architektur, konsequenter
Multi-User-Isolation und einem durchdachten Offline-Konzept. Die Sicherheitslage ist erfreulich:
keine SQL-Injection, kein XSS-Risiko, korrekte Auth-Logik. Die drei drängendsten Themenfelder
sind (1) stille Datenverluste im PWA-Offline-Pfad, (2) ein kaputter Fokusindikator, der die
Tastaturbedienbarkeit für betroffene Ansichten komplett bricht, und (3) ein Migrations-Bug,
der `updated_at` in der Datenbank nie aktualisiert. Quer durch den Code gibt es außerdem
erhebliche Design-Token-Verschuldung (hardcodierte RGBA-Werte) und eine Reihe von
Deutsch-Verstößen gegen den Apple Style Guide.

## P0 — Sofortmaßnahmen (vor nächstem Deployment)

### Datenverlust / PWA

**P0-1 — Outbox: 4xx-Antworten vernichten Daten stillschweigend**
`db.js:81–83` — Jeder 4xx-Fehler (z. B. 422 Validation, 404 Not Found) beim Outbox-Drain
löscht den Eintrag ohne Benutzerhinweis. Gespeicherte Buchungen gehen dauerhaft verloren.
Fix: 4xx-Einträge in eine `failed`-Queue verschieben und dem Nutzer eine Fehlermeldung anzeigen
statt `done++`.

**P0-2 — CSV-Import im Offline-Modus: Datei geht verloren**
`sw.js:131–133` — `request.clone().json()` schlägt bei `multipart/form-data` fehl; `body: null`
wird in die Outbox geschrieben. Beim Drain sendet der Server eine leere POST-Anfrage — Datei
permanent verloren.
Fix: Content-Type erkennen; wenn nicht JSON → sofort 503 zurückgeben statt 202.

### Barrierefreiheit

**P0-3 — Kein sichtbarer Fokusindikator (Tastatur-Blockade)**
`styles.css:2568, 2705` — `outline: var(--focus-ring)` ist ungültiges CSS (`--focus-ring`
enthält einen `box-shadow`-Wert, keinen `outline`-Wert). Tastaturnutzer haben in
Report-Transaktionszeilen und Forecast-Tabellenzeilen keinerlei Fokus-Sichtbarkeit.
Fix: `outline:` durch `box-shadow:` ersetzen.

### Datenbank

**P0-4 — `ON UPDATE CURRENT_TIMESTAMP` wird nie emittiert**
`migrations/0004_user_settings.py:45` — `server_onupdate=sa.text("CURRENT_TIMESTAMP")` ist in
SQLAlchemy ein No-Op; MariaDB erhält nie das DDL-Clause. `updated_at` wird ausschließlich durch
ORM-Writes aktualisiert, nicht durch direkte SQL-Updates.
Fix: `server_default=sa.text("CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP")` verwenden
+ neue Alembic-Revision.

**P0-5 — Fehlender Index auf `transactions.category_id`**
`models.py` — Ohne expliziten Index führt jedes Kategorie-DELETE zu einem Full Table Scan
auf `transactions` (InnoDB FK-Check). Wächst bei vielen Buchungen zum Performance-Problem.
Fix: `Index("ix_transactions_category_id", "category_id")` in Modell + Alembic-Migration
ergänzen.

### Lizenz-Compliance

**P0-6 — OFL-Lizenztext fehlt bei den Fonts**
`frontend/fonts/` — Die SIL Open Font License schreibt vor, dass der Lizenztext die Fonts bei
der Weitergabe begleitet. Ohne `OFL.txt` verletzt das Deployment die Lizenzbedingungen.
Fix: `frontend/fonts/OFL.txt` mit dem originalen SIL OFL 1.1-Text hinzufügen.

## P1 — Kurzfristig (nächster Sprint)

### Barrierefreiheit

**P1-1 — Fehlende `:focus-visible`-Regeln auf interaktiven Elementen**
`styles.css` — `.fab`, `.hamburger-btn`, `.drawer-close-btn`, `.drawer-nav-item`, `.type-btn`,
`.submit-btn`, `.segmented button` haben keine `:focus-visible`-Styles.

**P1-2 — ARIA-Attribute unvollständig**
`index.html:192` — Drawer fehlt `aria-modal="true"` auf `role="dialog"`. Statische
`.modal`-Elemente fehlen `role="dialog"` und `aria-modal="true"`. `index.html:665–668` —
`role="tab"`-Buttons haben kein initiales `aria-selected`.

### Design-Tokens (dringlichste Verstöße)

**P1-3 — Theme-Breaks: hardcodierte RGBA-Werte**
- `styles.css:1081, 1347` — Overlay `rgba(20,14,8,0.35)` nicht theme-aware → neues Token `--overlay-bg`
- `styles.css:1084–1085` — `backdrop-filter: blur(4px)` hardcodiert → `--blur-overlay` Token
- `styles.css:1392–1393` — Modal `::before`-Spiegelglanz `rgba(255,255,255,0.45)` → `--specular-highlight`
- `styles.css:1509, 1516` und weitere — `rgba(255,255,255,0.35)` als Hairline auf Buttons → `var(--hairline)`
- `styles.css:1919–1920, 1929–1930` — Submit-Button-Schatten → `var(--hairline)` + `var(--edge-dark)`
- `styles.css:2031, 2033` — Save-Button-Border/-Schatten → `var(--hairline)`
- `styles.css:2787–2788` — Toast `box-shadow` mit Raw-RGBA → `var(--shadow-elevated)`

**P1-4 — Drei-Wege-Inkonsistenz beim Swipe-Action-Offset**
`styles.css:751` — `transform: translateX(-92px)` hardcodiert; Token `--swipe-action-w: 92px`
existiert; JS-Konstante `SWIPE_ACTION_WIDTH = 88` weicht ab. Alle drei Stellen vereinheitlichen.

### UI-Texte (Apple Style Guide)

**P1-5 — Kritische Grammatik- und Sprachfehler**
- `app.js:638` — Fehlendes Komma: „Tippe auf +, um eine hinzuzufügen."
- `app.js:2364` — Passiv-Jargon „sind Pflichtfelder" → „Gib Betrag und Datum ein."
- `app.js:3289` — „Details in der Browser-Console" → „${r.errors.length} Zeilen übersprungen."

**P1-6 — Fehlerhafte Komposita (überflüssige Bindestriche)**
`index.html:209, 216, 241, 423, 432` —
- „Kategorien-Übersicht" → „Kategorienübersicht"
- „Buchungs-Übersicht" → „Buchungsübersicht"
- „Kategorien-Analyse" → „Kategorienanalyse"
- „Tag-Analyse" → „Taganalyse"

**P1-7 — Entwickler-Terminologie im Nutzer-UI**
- `index.html:514` — „SW-Status" → „Service-Worker-Status"
- `index.html:518` — „Offline-Outbox" → „Ausstehende Änderungen"
- `index.html:580` — „User-Agent" → „User Agent" oder „Browser-Kennung"

### Backend

**P1-8 — Tag-Normalisierung zerstört Groß-/Kleinschreibung**
`app.js:499` — `normalizeTx()` wandelt Tags beim Lesen in Kleinbuchstaben um. Tags, die via
CSV mit Mischschreibung importiert wurden, werden bei jedem Edit dauerhaft mutiert.
Fix: Normalisierung nur beim Vergleich, nicht beim Lesen.

**P1-9 — CSV-Fehler leakt Datenbankstruktur**
`crud.py:610` — `e.orig` enthält den rohen DB-Fehler inkl. Tabellen-/Spaltennamen und wird
direkt an den API-Client zurückgegeben.
Fix: Generische Fehlermeldung zurückgeben, Detail nur ins Server-Log schreiben.

**P1-10 — `TransactionOut` verletzt Backend-Konvention**
`schemas.py:148–160` — `TransactionOut` nutzt `serialization_alias` ohne
`populate_by_name=True`, asymmetrisch zu `TransactionIn` und der dokumentierten Konvention.

## Was gut ist

- **Sicherheitsarchitektur** — Konsequente `user_id`-Filterung in allen Queries, timing-sicherer Header-Check, keine SQL-Injection, kein XSS-Risiko durch durchgängiges `_escText()`/`_escAttr()`.
- **Offline-Grundgerüst** — Network-first für API, cache-first für Assets, 401-Cache-Drop korrekt implementiert.
- **Asset-Selbsthosting** — Keine CDN-Abhängigkeiten, alle Fonts/Icons/JS lokal. Privacy-Ziel konsequent umgesetzt.
- **Design-Token-System** — Umfangreich und strukturiert; die meisten Komponenten nutzen es korrekt.
- **Subagent-Infrastruktur** — Die 8 Review-Agents in `.claude/agents/` sind ein herausragendes Werkzeug für dauerhaft hohe Codequalität.
- **Backend-Konventionen** — Alembic-Migrationen, Pydantic v2, Schema-Trennung In/Out, StaticFiles-Mount-Reihenfolge korrekt eingehalten.
